# T'SUKI sur la mer ── ホームページ公開ファイル一式

## 構成（リポジトリ直下に置く）
- index.html … トップページ本体
- site.jsx … ページの中身（React）
- fairs.json … ブライダルフェアのデータ（カレンダーに表示される）
- Reservation.html … 予約フォームページ
- tweaks-panel.jsx … 補助スクリプト
- favicon.svg … タブのアイコン
- assets/ … 画像・動画

## Vercel 公開手順
1. GitHub の `tsuki-surlamer` リポジトリを開く
2. 「Add file」→「Upload files」
3. このフォルダの【中身を全部】ドラッグ＆ドロップ（assets フォルダごと）
4. 「Commit changes」
5. Vercel 側の Root Directory は【空欄（直下）】に設定 → Deploy
6. 数十秒で公開

## フェアの更新方法（fairs.json を直接編集）
fairs.json を開いて、1件＝1開催日の形式で編集します。

例（1件分）:
{
  "date": "2026-08-01",        ← 開催日（YYYY-MM-DD）
  "d": "8 / 1",                ← 表示用（月 / 日）
  "w": "SAT",                  ← 曜日（SUN〜SAT）
  "name": "夏の試食フェア",      ← タイトル（自由）
  "time": "10:00 / 14:00",     ← 時間
  "tags": ["試食付き", "人気No.1"],  ← 内容タグ（自由・複数可）
  "hot": true,                 ← true にすると最初のタグが金色で強調
  "slug": "summer-tasting"     ← 予約リンク用のID（英数字で任意・重複させない）
}

- 上記の { … } を [ ] の中にカンマ区切りで並べます
- 同じフェアを複数日やる場合は、date だけ変えた要素を日数分ぶん並べます
  （name・time・tags・slug は同じでOK）
- 編集後、GitHub で fairs.json を上書きアップロード → 数十秒で反映

※ カレンダーは実際のカレンダー上の曜日で表示されます。"w" は予約ページ等での表示用です。
  日付と曜日がずれないよう、date を基準に入力してください。
